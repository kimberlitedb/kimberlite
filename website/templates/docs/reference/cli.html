CLI Reference

Complete reference for the kimberlite command-line interface.

Overview

The Kimberlite CLI provides commands for initializing, running, and interacting with your database.

$ kimberlite --help Kimberlite unified CLI - the compliance-first database Usage: kimberlite <COMMAND> Commands: version Show version information init Initialize a new data directory start Start the Kimberlite server repl Interactive SQL REPL query Execute a single SQL query stream Stream management commands info Show server information help Print this message or help for subcommands

kimberlite init

Initialize a new data directory with required files and structure.

kimberlite init [OPTIONS] <PATH>

Arguments

Argument Description
<PATH> Path to the data directory to create

Options

Option Description
--development Enable development mode (disables fsync for faster writes)

Example

$ kimberlite init ./data --development

Created Structure

./data/
├── config.toml      # Server configuration
├── log/             # Append-only log directory
├── store/           # Storage files
└── projections.db   # B+tree projection store

kimberlite start

Start the Kimberlite server daemon.

kimberlite start [OPTIONS] <PATH>

Arguments

Argument Description
<PATH> Path to the data directory

Options

Option Default Description
-a, --address 5432 Address to bind to (port or full address)
--development - Disable replication for development

Examples

# Start on default port (5432) $ kimberlite start ./data # Start on custom port $ kimberlite start --address 3000 ./data # Start with full address $ kimberlite start --address 0.0.0.0:3000 ./data

kimberlite repl

Start an interactive SQL REPL session.

kimberlite repl [OPTIONS]

Options

Option Default Description
-a, --address 127.0.0.1:5432 Server address to connect to
-t, --tenant 1 Tenant ID

REPL Commands

Command Description
.help Show help message
.tables List all tables
.exit, .quit Exit the REPL

kimberlite query

Execute a single SQL query.

kimberlite query [OPTIONS] <SQL>

Arguments

Argument Description
<SQL> SQL query string

Options

Option Default Description
-s, --server 127.0.0.1:5432 Server address
-t, --tenant 1 Tenant ID
-a, --at - Query at a specific log position (time travel)

Examples

# Simple query $ kimberlite query "SELECT * FROM patients" # Time-travel query at position 42 $ kimberlite query --at 42 "SELECT * FROM patients"

kimberlite stream

Manage event streams.

stream create

Create a new event stream.

kimberlite stream create [OPTIONS] <NAME>
Option Default Description
-c, --class non-phi Data classification: non-phi, phi, or deidentified

stream append

Append events to a stream.

kimberlite stream append <STREAM_ID> <EVENTS>...

stream read

Read events from a stream.

kimberlite stream read [OPTIONS] <STREAM_ID>
Option Default Description
-f, --from 0 Starting offset
-m, --max-bytes 65536 Maximum bytes to read

kimberlite version

Display version and build information.

$ kimberlite version kimberlite-cli 0.1.0 The compliance-first database for regulated industries. Build info: Rust version: 1.88+ Target: aarch64 OS: macos
