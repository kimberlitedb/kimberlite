Quick Start

Get Kimberlite running in 90 seconds.

1. Download

Download the pre-built binary for your platform:

Terminal 
$ curl -Lo kimberlite.zip https://kimberlite.dev/download && unzip kimberlite.zip

Or build from source:

Terminal 
$ cargo install kimberlite-cli

2. Initialize

Create a data directory for your database:

Terminal 
$ ./kimberlite init ./data --development
Initializing in DEVELOPMENT mode (relaxed durability)

Data directory initialized successfully!

  Path:   /path/to/data
  Config: ./data/config.toml

Note: The --development flag disables fsync for faster writes during development. Remove it for production deployments.

3. Start the Server

Start the Kimberlite server:

Terminal 
$ ./kimberlite start --address 3000 ./data
Kimberlite - the compliance-first database

  Data directory: /path/to/data
  Bind address:   127.0.0.1:3000
  Mode:           development (no replication)

Server is ready. Press Ctrl+C to stop.

4. Connect and Query

In a new terminal, connect with the interactive REPL:

Terminal 
$ ./kimberlite repl --address 127.0.0.1:3000
Kimberlite SQL REPL
Connected to: 127.0.0.1:3000
Tenant ID:    1

Type .help for help, .exit to quit.

Try some SQL commands:

kimberlite-cli 
kimberlite> CREATE TABLE patients (id BIGINT, name TEXT);
(0 rows)

kimberlite> INSERT INTO patients VALUES (1, 'Jane Doe');
(1 row)

kimberlite> SELECT * FROM patients;
 id | name
----+----------
  1 | Jane Doe

(1 row)
