What Enables Determinism?

No IO — Reading from disk/network is non-deterministic
No clocks — Time changes between executions
No randomness — Random values differ each run
Pure functions — Math doesn't lie

Why Determinism Matters

Replication — Replicas stay in sync by replaying commands
Testing — Tests are reproducible, no flakiness
Time travel — Replay to any point in history
Debugging — Reproduce exact conditions that caused a bug
